# Deterministic Actions And Daily Command Center Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Telegram button actions deterministic and add a Daily Command Center endpoint that returns the four daily commitments OpenClue should render every morning.

**Architecture:** LifeOS API owns durable action execution and daily plan orchestration. OpenClue sends callback values or asks for the command center, then renders the API result instead of hand-routing button logic through prompt instructions. Existing domain endpoints remain available, but the new orchestration endpoints become the stable Telegram-facing contract.

**Tech Stack:** FastAPI, SQLAlchemy 2, Pydantic v2, Alembic-free JSON/state reuse, pytest, OpenClaw contract rendering.
